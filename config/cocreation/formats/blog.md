# Formato — Artigo de Blog

## Estrutura de saída

1. **Intenção de busca** — qual pergunta ou necessidade do leitor este artigo resolve (informacional,
   comparativo, transacional).
2. **Persona-alvo**.
3. **Título editorial** — título voltado para o leitor humano, atrativo e claro.
4. **Título SEO** — versão otimizada para busca, podendo diferir levemente do título editorial se
   necessário para incluir termo de busca relevante.
5. **Slug** — versão em URL (minúsculo, hífens, sem acento, sem stopword desnecessária).
6. **Meta description** — até ~155 caracteres, resume o artigo e inclui motivo para clicar.
7. **Headings (H2/H3)** — estrutura completa do artigo em tópicos, organizada em progressão lógica
   (contexto → desenvolvimento → aplicação → conclusão).
8. **Texto completo** — desenvolvido por heading, com parágrafos objetivos, exemplos quando cabível
   (sempre dentro das regras de `references/factuality.md`), sem enrolação para bater contagem de
   palavras.
9. **Referências** — fontes usadas (documentação interna, pesquisa aprovada) quando aplicável.
10. **Links internos sugeridos** — outros conteúdos do mesmo domínio/segmento que fariam sentido
    linkar a partir deste artigo (sugestão de tema/URL-alvo, não necessariamente URL real se não
    existir ainda).
11. **CTA final** — convite consultivo coerente com a marca (diagnóstico, material, conversa com
    especialista).

## Regras

- Título SEO não pode ser clickbait desconectado do conteúdo real do artigo.
- Headings devem formar um índice que faz sentido sozinho, como sumário.
- Todo dado ou número no corpo do artigo segue a classificação de `references/factuality.md`.
- Evitar parágrafos de introdução genéricos ("Neste artigo, vamos explorar...") — ir direto ao
  contexto relevante.
